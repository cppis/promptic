## 2. 기술 아키텍처

### 2.1 아키텍처

```
Claude Desktop / Claude Code
    │
    │  MCP (stdio, JSON-RPC)
    │
    ▼
┌─────────────────────────────────────────────┐
│  PromptLens MCP Server (v0.4.0)             │
│                                             │
│  index.js (16개 도구 등록 + 라우팅)           │
│    │                                        │
│    ├── analyzer.js                          │
│    │   ├── analyzePrompt()     ← local 모드  │
│    │   └── analyzeWithApi()   ← api 모드     │
│    │                                        │
│    ├── storage.js                           │
│    │   ├── _loadIndex()        ← 프로젝트 목록 │
│    │   ├── _loadProject(id)   ← 프로젝트 파일 │
│    │   ├── _saveProject(id)   ← 원자적 저장   │
│    │   └── _migrateIfNeeded() ← 1회 마이그레이션 │
│    │                                        │
│    ├── importer.js                          │
│    │   ├── importClaudeDesktop() ← .json    │
│    │   └── importClaudeCode()    ← .jsonl   │
│    │                                        │
│    ├── exporter.js                          │
│    │   ├── toJson()  toMarkdown()  toCsv()  │
│    │   └── fromJson()                       │
│    │                                        │
│    ├── query.js                             │
│    │   ├── filterEntries()                  │
│    │   └── sortEntries()                    │
│    │                                        │
│    ├── differ.js    ← word-level diff (LCS) │
│    └── visualizer.js ← Chart.js 대시보드    │
│                                             │
│  ~/.promptlens/                             │
│    projects/_index.json                     │
│    projects/{id}.json                       │
│    exports/   snapshots/   settings.json    │
│    dashboard-*.html                         │
└─────────────────────────────────────────────┘
         │
         │ HTTPS (api 모드만, 사용자 API 키)
         ▼
    Claude API (Anthropic)
```

### 2.2 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|---------|
| MCP 런타임 | @modelcontextprotocol/sdk | 공식 SDK, stdio transport |
| 스키마 검증 | Zod | 도구 파라미터 타입 안전성 |
| AI 분석 | Claude API (fetch) | BYOK, 서버 불필요 |
| 저장소 | 프로젝트별 JSON 파일 (fs) | 프로젝트 단위 I/O, 백업/공유 용이 |
| 테스트 | node:test (내장) | 외부 의존성 없음 |
| 런타임 | Node.js 18+ | ES Modules, 내장 fetch |

### 3.2 핵심 구현 사항

**storage.js — 저장소 구조 전환:**

```
변경 전:
  data.json 전체 로드 → 수정 → 전체 저장

변경 후:
  _loadIndex()          → projects/_index.json 로드 (프로젝트 목록)
  _loadProject(id)      → projects/{id}.json 로드 (필요한 프로젝트만)
  _saveProject(id, data)→ temp 파일 → rename (원자적 저장)
  _migrateIfNeeded()    → data.json 감지 시 1회 자동 분리
```

**exporter.js — 직렬화 엔진:**

```javascript
toJson(project, entries)     // → .promptlens.json (완전 복원 가능)
toMarkdown(project, entries) // → Markdown 리포트
toCsv(project, entries)      // → CSV (axisScores 5개 컬럼 플래트닝)
fromJson(content)            // → { project, history, meta, warnings }
```

**query.js — 필터 엔진:**

```javascript
filterEntries(entries, filter) // score / grade / date / tags(AND/OR) / missing / versionsOnly / search
sortEntries(entries, sortBy)   // score_asc/desc, date_asc/desc
summarizeFilter(filter)        // 사람이 읽기 좋은 필터 요약 문자열
```

## 4. v0.5.0 개발 계획

### 4.1 목표

두 가지 신규 기능으로 PromptLens의 핵심 가치를 강화한다.

| 기능 | 목표 | 핵심 가치 |
|------|------|----------|
| AI 해석 시뮬레이션 | AI 1인칭 서술로 해석 과정 체험 | 진입 장벽 최소화, "아하 모먼트" |
| 누적 패턴 코칭 | 반복 약점 추출 + 개인화 행동 제안 | 일회성 → 지속적 성장 도구 |

### 4.2 구현 일정 (5주)

| 주차 | 단계 | 산출물 |
|------|------|--------|
| 1주 | AI 해석 시뮬레이션 — local 모드 | analyzer.js에 narrative 생성 로직 (템플릿 기반 서술문), analyze_prompt 응답에 `narrative` 필드 추가 |
| 2주 | AI 해석 시뮬레이션 — api 모드 | Claude API 시스템 프롬프트 확장 (1인칭 서술 지시), api 응답에서 narrative 파싱, local/api 서술 품질 비교 테스트 |
| 3주 | 누적 패턴 코칭 — 분석 엔진 | coaching.js 신규 모듈 (축별 통계, Missing 패턴 빈도, 도메인별 교차분석, 성장 추이, 코칭 메시지 템플릿) |
| 4주 | 누적 패턴 코칭 — MCP 도구 | `get_coaching` 도구 등록 (period, projectId), 최소 데이터 검증 (10개 미만 안내), 테스트 코드 |
| 5주 | 통합 테스트 + 문서 | narrative + coaching E2E 테스트, 문서 업데이트, v0.5.0 릴리스 |

### 4.3 도구 변경

| 유형 | 도구 | 내용 |
|------|------|------|
| 수정 | `analyze_prompt` | 응답에 `narrative` 필드 추가 |
| 신규 | `get_coaching` | 누적 패턴 분석 + 개인화 코칭 리포트 |

도구 총 수: 16개 → **17개**
