# Promptic — 사용 샘플 모음 (v1.1)

> 실제 대화 흐름을 기반으로 한 사용 시나리오
> 각 샘플은 독립 실행 가능하며, 앞 샘플의 결과를 이어받는 형태로도 사용 가능

---

## 샘플 1. 첫 시작 — 설정 및 프로젝트 생성

### 1-1. 현재 설정 확인

**사용자:**
```
설정 확인해줘
```

**도구 호출:** `get_settings`

**예상 응답:**
```
현재 설정
─────────────────────────────
API 키:        미등록 (local 모드만 사용 가능)
모델:          claude-haiku-4-5
활성 프로젝트:  없음
저장 경로:     ~/.prompatic/data
```

---

### 1-2. API 키 등록

**사용자:**
```
API 키 등록해줘. sk-ant-api03-...
```

**도구 호출:** `set_api_key`

```json
{
  "apiKey": "sk-ant-api03-...",
  "model": "claude-haiku-4-5"
}
```

**예상 응답:**
```
✅ API 키가 등록되었습니다.
모델: claude-haiku-4-5
api 모드 및 deep 모드 사용 가능합니다.
```

---

### 1-3. 프로젝트 생성 및 활성화

**사용자:**
```
"게임 서버 프롬프트" 프로젝트 만들어줘
```

**도구 호출:** `create_project`

```json
{ "name": "게임 서버 프롬프트" }
```

**사용자:**
```
방금 만든 프로젝트 활성화해줘
```

**도구 호출:** `set_active_project`

```json
{
  "action": "set",
  "projectName": "게임 서버 프롬프트"
}
```

**예상 응답:**
```
✅ "게임 서버 프롬프트" 프로젝트가 활성화되었습니다.

## 샘플 2. 기본 분석 — local 모드

### 2-1. 초안 프롬프트 분석

유저가 Go 서버 코드 리뷰 요청 프롬프트를 처음 작성한 상황.

**사용자:**
```

---
```

**도구 호출:** `analyze_prompt`

```json
{
  "prompt": "코드 리뷰해줘.",
  "mode": "local"
}
```

**예상 응답:**
```

**사용자:**
```

## 샘플 3. API 모드 — 3색 리포트

**도구 호출:** `analyze_prompt`

```json
{
  "prompt": "Unity 게임에서 플레이어 이동 시스템을 구현해줘...",
  "mode": "api"
}
```

**예상 응답:**
```

─────────────────────────────
종합 점수: 61 / 100   등급: C

  clarity        ██████░░░░  62
  completeness   ██████░░░░  58
  specificity    ██████░░░░  60
  context        █████░░░░░  55
  output_format  ███████░░░  68

3색 요소 분류:

  ✅ Referenced (명시된 요소)
     - 입력 방식: WASD
     - 물리 기반 이동
     - 벽 충돌 처리
     - 언어: C#

  🔶 Inferred (추론된 요소)
     - Unity 버전: 현재 LTS 추정
     - 이동 방식: CharacterController 또는 Rigidbody
     - 카메라 연동 여부: 불명확

  ❌ Missing (누락된 요소)
     - 이동 속도/점프 여부 미지정
     - Update vs FixedUpdate 물리 처리 기준 없음
     - 스크립트 구조 (MonoBehaviour 단독 vs 컴포넌트 분리) 미지정
     - 모바일/PC 구분 없음

개선 제안:
  "Unity 2022 LTS, C#. FixedUpdate 기반 Rigidbody 이동.
   이동 속도: 5f, 점프 없음. 컴포넌트 분리 (InputHandler / MovementController).
   PC 전용."

entryId: `h_p1q2r3s4t5_u6v7w8`
```

## 샘플 5. 자동 개선 (`>> fix`)

> Feature 1 구현 후 사용 가능 (API 키 필수)

**사용자:**
```
>> fix
```

내부적으로 직전 `entryId`의 분석 결과(`missingElements`)를 기반으로 Claude API가 개선된 프롬프트를 생성한다.

**예상 응답:**
```
자동 개선 생성
─────────────────────────────
원본 점수: 61 (C)  →  예상 개선 점수: 85+ (A)

개선된 프롬프트:
─────────────────────────────
당신은 Unity 2022 LTS C# 전문가입니다.
아래 요구사항에 맞는 플레이어 이동 시스템을 구현해주세요.

요구사항:
- 입력: WASD (Input System 패키지 사용)
- 물리: Rigidbody, FixedUpdate에서 처리
- 이동 속도: 5f (직렬화 가능하도록 [SerializeField] 사용)
- 벽 충돌: Physics 레이어 기반 LayerMask
- 플랫폼: PC 전용

컴포넌트 설계:
- InputHandler.cs: 입력 수신 담당
- PlayerMovement.cs: 물리 이동 담당 (InputHandler 의존성 주입)

출력: 두 파일의 전체 코드 + 각 컴포넌트 역할 주석
─────────────────────────────

변경 요약:
  추가: 페르소나, Unity 버전, Input System 명시
  추가: FixedUpdate 물리 기준 명시
  추가: 속도값 구체화 ([SerializeField])
  추가: 컴포넌트 분리 구조 명시
  추가: 출력 형식 (파일 2개 + 주석)

저장하려면 ">> fix+save" 를 입력하세요.
```

**사용자:**
```
이 세 가지 프롬프트 버전을 한 번에 비교 분석해줘

버전 A: "번역해줘."
버전 B: "다음 한국어 텍스트를 영어로 번역해줘."

## 빠른 참조 — 트리거 단축 명령

| 트리거 | 동작 | 전제 조건 |
|--------|------|----------|
| `>> 분석` / `>> anz` | local 모드 분석 | — |
| `>> 정밀분석` / `>> deep` | deep 모드 분석 | API 키, parentId |
| `>> re` / `>> v2` | 직전 entryId로 재분석 | 세션 내 이전 분석 |
| `>> fix` | 자동 개선 생성 | API 키 |
| `>> fix+save` | 자동 개선 + 히스토리 저장 | API 키 |
